Saya ada bug navigasi mobile: di viewport mobile, pas sidebar dibuka muncul overlay hitam, tapi overlay itu (atau sesuatu yang layer-nya di atasnya) nge-block semua klik — saya gak bisa tap link sidebar, tombol close, atau nav items. Yang kejadian malah balik ke halaman sebelumnya. Ini normal-normal aja di desktop.
Tolong cek logic toggle sidebar di komponen React (kemungkinan di Sidebar.tsx, MobileHeader.tsx, atau file mana pun yang megang state sidebarOpen dan class .overlay / .sidebar) buat masalah spesifik ini:
Bug sinkronisasi state: pastikan class visible di overlay dan class open di sidebar di-toggle bareng, secara atomic, dari satu state variable yang sama — bukan dua useState terpisah yang bisa gak sinkron.
Timing display vs opacity: CSS overlay-nya pakai display: none sebagai default dan .overlay.visible { opacity: 1 } buat transisi fade. Pastikan JS set display (lewat inline style atau class mounting) SEBELUM toggle visible buat transisi opacity — bug yang sering kejadian itu overlay udah ada di DOM dengan opacity: 0 tapi masih pointer-events: auto, yang secara gak keliatan nge-block semua klik di bawahnya sebelum fade-in-nya selesai.
Tambahin kontrol pointer-events yang eksplisit: pastikan .overlay { pointer-events: none } sebagai default dan .overlay.visible { pointer-events: auto } pas beneran ditampilin, supaya overlay yang transparan/lagi transisi gak pernah nyegat tap.
Handler klik overlay: pastikan onClick di overlay cuma nutup sidebar (onClick={() => setSidebarOpen(false)}) dan TIDAK trigger navigasi, history.back(), atau redirect router apapun.
Event bubbling: cek apakah tap di link sidebar dalam drawer itu bubbling naik ke handler klik overlay di bawahnya dan nutup/navigasi duluan sebelum onClick link-nya sendiri kejalan — kalau iya, tambahin e.stopPropagation() di container drawer sidebar.
Tolong review kode state management sidebar/overlay yang sebenarnya dan fix akar masalahnya dari poin-poin di atas, terus konfirmasi fix-nya beneran jalan khusus di viewport mobile (≤768px), bukan cuma di desktop.
